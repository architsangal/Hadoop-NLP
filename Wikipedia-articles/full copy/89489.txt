The notation a ≠ b means that a is not equal to b. It does not say that one is greater than the other, or even that they can be compared in size. If the values in question are elements of an ordered set, such as the integer s or the real number s, they can be compared in size. The notation a &lt; b means that a is less than b. The notation a &gt; b means that a is greater than b. In either case, a is not equal to b. These relations are known as strict inequalities. The notation a &lt; b may also be read as &quot; a is strictly less than b &quot;. In contrast to strict inequalities, there are two types of inequality relations that are not strict: The notation a ≤ b means that a is less than or equal to b (or, equivalently, not greater than b, or at most b). The notation a ≥ b means that a is greater than or equal to b (or, equivalently, not less than b, or at least b) An additional use of the notation is to show that one quantity is much greater than another, normally by several orders of magnitude. [ ≪ ] [ ≫ ] The notation a b means that a is much less than b. (In measure theory, however, this notation is used for absolute continuity, an unrelated concept.) The notation a b means that a is much greater than b. [ Properties ] Inequalities are governed by the following properties. Note that, for the transitivity, converse, addition and subtraction, and multiplication and division properties, the property also holds if strict inequality signs ( &lt; and &gt; ) are replaced with their corresponding non-strict inequality sign (≤ and ≥). [ Transitivity ] The Transitive property of inequality states: If a &gt; b and b &gt; c, then a &gt; c. If a &lt; b and b &lt; c, then a &lt; c. If a &gt; b and b = c, then a &gt; c. If a &lt; b and b = c, then a &lt; c. [ Converse ] The relations &lt; and &gt; are each other&apos;s converse : If a &lt; b, then b &gt; a. If a &gt; b, then b &lt; a. [ Addition and subtraction ] The properties that deal with addition and subtraction state: If a &lt; b, then a + c &lt; b + c and a − c &lt; b − c. If a &gt; b, then a + c &gt; b + c and a − c &gt; b − c. i.e., the real numbers are an ordered group under addition. [ Multiplication and division ] The properties that deal with multiplication and division state: If c is positive and a &lt; b, then ac &lt; bc and a/c &lt; b/c. If c is negative and a &lt; b, then ac &gt; bc and a/c &gt; b/c. More generally this applies for an ordered field, see below. [ Additive inverse ] The properties for the additive inverse state: If a &lt; b then &amp; minus;a &gt; &amp; minus;b. If a &gt; b then &amp; minus;a &lt; &amp; minus;b. [ Multiplicative inverse ] The properties for the multiplicative inverse state: If a &lt; b then 1/a &gt; 1/b. If a &gt; b then 1/a &lt; 1/b. If a &lt; b then 1/a &lt; 1/b. If a &gt; b then 1/a &gt; 1/b. [ Applying a function to both sides ] The graph of &apos;&apos;y&apos;&apos; = ln &apos;&apos;x&apos;&apos; Any strictly monotonic ally increasing function may be applied to both sides of an inequality (provided they are in the domain of that function) and it will still hold. Applying a strictly monotonically decreasing function to both sides of an inequality means the opposite inequality now holds. The rules for additive and multiplicative inverses are both examples of applying a monotonically decreasing function. For a non-strict inequality (a ≤ b, a ≥ b): Applying a monotonically increasing function preserves the relation (≤ remains ≤, ≥ remains ≥) Applying a monotonically decreasing function reverses the relation to its converse (≤ becomes ≥, ≥ becomes ≤) As an example, consider the application of the natural logarithm to both sides of an inequality: 0 &lt; a &lt; b \Leftrightarrow \ln(a) &lt; \ln(b). &lt; /math &gt; This is true because the natural logarithm is a strictly increasing function. ==Ordered fields== If (&apos;&apos;F&apos;&apos;, +, &amp; times;) is a [[Field (mathematics)|field]] and ≤ is a [[total order]] on &apos;&apos;F&apos;&apos;, then (&apos;&apos;F&apos;&apos;, +, &amp; times;, ≤) is called an [[ordered field]] if and only if: * &apos;&apos;a&apos;&apos; ≤ &apos;&apos;b&apos;&apos; implies &apos;&apos;a&apos;&apos; + &apos;&apos;c&apos;&apos; ≤ &apos;&apos;b&apos;&apos; + &apos;&apos;c&apos;&apos;; * 0 ≤ &apos;&apos;a&apos;&apos; and 0 ≤ &apos;&apos;b&apos;&apos; implies 0 ≤ &apos;&apos;a&apos;&apos; &amp; times; &apos;&apos;b&apos;&apos;. Note that both (&apos;&apos;&apos;Q&apos;&apos;&apos;, +, &amp; times;, ≤) and (&apos;&apos;&apos;R&apos;&apos;&apos;, +, &amp; times;, ≤) are [[ordered field]]s, but ≤ cannot be defined in order to make (&apos;&apos;&apos;C&apos;&apos;&apos;, +, &amp; times;, ≤) an [[ordered field]], because &amp; minus;1 is the square of &apos;&apos;i&apos;&apos; and would therefore be positive. The non-strict inequalities ≤ and ≥ on real numbers are [[total order]] s. The strict inequalities &lt; and &gt; on real numbers are [[Total_order#Strict_total_order|strict total orders]]. == Chained notation == The notation &apos;&apos;&apos;&apos;&apos;a&apos;&apos; &lt; &apos;&apos;b&apos;&apos; &lt; &apos;&apos;c&apos;&apos;&apos;&apos;&apos; stands for &quot; &apos;&apos;a&apos;&apos; &lt; &apos;&apos;b&apos;&apos; and &apos;&apos;b&apos;&apos; &lt; &apos;&apos;c&apos;&apos; &quot;, from which, by the transitivity property above, it also follows that &apos;&apos;a&apos;&apos; &lt; &apos;&apos;c&apos;&apos;. Obviously, by the above laws, one can add/subtract the same number to all three terms, or multiply/divide all three terms by same nonzero number and reverse all inequalities according to sign. Hence, for example, &apos;&apos;a&apos;&apos; &lt; &apos;&apos;b&apos;&apos; + &apos;&apos;e&apos;&apos; &lt; &apos;&apos;c&apos;&apos; is equivalent to &apos;&apos;a&apos;&apos; − &apos;&apos;e&apos;&apos; &lt; &apos;&apos;b&apos;&apos; &lt; &apos;&apos;c&apos;&apos; − &apos;&apos;e&apos;&apos;. This notation can be generalized to any number of terms: for instance, &apos;&apos;&apos;&apos;&apos;a&apos;&apos; &lt; sub &gt; 1 &lt; /sub &gt; ≤ &apos;&apos;a&apos;&apos; &lt; sub &gt; 2 &lt; /sub &gt; ≤... ≤ &apos;&apos;a&apos;&apos; &lt; sub &gt; &apos;&apos;n&apos;&apos; &lt; /sub &gt; &apos;&apos;&apos; means that &apos;&apos;a&apos;&apos; &lt; sub &gt; &apos;&apos;i&apos;&apos; &lt; /sub &gt; ≤ &apos;&apos;a&apos;&apos; &lt; sub &gt; &apos;&apos;i&apos;&apos;+1 &lt; /sub &gt; for &apos;&apos;i&apos;&apos; = 1, 2,..., &apos;&apos;n&apos;&apos;   &amp; minus;  1. By transitivity, this condition is equivalent to &apos;&apos;a&apos;&apos; &lt; sub &gt; &apos;&apos;i&apos;&apos; &lt; /sub &gt; ≤ &apos;&apos;a&apos;&apos; &lt; sub &gt; &apos;&apos;j&apos;&apos; &lt; /sub &gt; for any 1 ≤ &apos;&apos;i&apos;&apos; ≤ &apos;&apos;j&apos;&apos; ≤ &apos;&apos;n&apos;&apos;. When solving inequalities using chained notation, it is possible and sometimes necessary to evaluate the terms independently. For instance to solve the inequality 4&apos;&apos;x&apos;&apos; &lt; 2&apos;&apos;x&apos;&apos; + 1 ≤ 3&apos;&apos;x&apos;&apos; + 2, it is not possible to isolate &apos;&apos;x&apos;&apos; in any one part of the inequality through addition or subtraction. Instead, the inequalities must be solved independently, yielding &apos;&apos;x&apos;&apos; &lt; 1/2 and &apos;&apos;x&apos;&apos; ≥ &amp; minus;1 respectively, which can be combined into the final solution &amp; minus;1 ≤ &apos;&apos;x&apos;&apos; &lt; 1/2. Occasionally, chained notation is used with inequalities in different directions, in which case the meaning is the [[logical conjunction]] of the inequalities between adjacent terms. For instance, &apos;&apos;a&apos;&apos; &lt; &apos;&apos;b&apos;&apos; = &apos;&apos;c&apos;&apos; ≤ &apos;&apos;d&apos;&apos; means that &apos;&apos;a&apos;&apos; &lt; &apos;&apos;b&apos;&apos;, &apos;&apos;b&apos;&apos; = &apos;&apos;c&apos;&apos;, and &apos;&apos;c&apos;&apos; ≤ &apos;&apos;d&apos;&apos;. [ Vector inequalities ] Inequality relationships similar to those defined above can also be defined for column vector. If we let the vectors x,y\in\mathbb{R}^n (meaning that x = \left(x_1,x_2,\ldots,x_n\right)^T and y = \left(y_1,y_2,\ldots,y_n\right)^T where x_i and y_i are real numbers for i=1,\ldots,n ), we can define the following relationships. x = y \ if x_i = y_i\ for i=1,\ldots,n x &lt; y \ if x_i &lt; y_i\ for i=1,\ldots,n x \leq y if x_i \leq y_i for i=1,\ldots,n and x \neq y x \leqq y if x_i \leq y_i for i=1,\ldots,n Similarly, we can define relationships for x &gt; y, x \geq y, and x \geqq y. We note that this notation is consistent with that used by Matthias Ehrgott in Multicriteria Optimization (see References). We observe that the property of Trichotomy (as stated above) is not valid for vector relationships. We consider the case where x = \left[ 2, 5 \right]^T and y = \left[ 3, 4 \right] ^T. There exists no valid inequality relationship between these two vectors. Also, a multiplicative inverse would need to be defined on a vector before this property could be considered. However, for the rest of the aforementioned properties, a parallel property for vector inequalities exists.
